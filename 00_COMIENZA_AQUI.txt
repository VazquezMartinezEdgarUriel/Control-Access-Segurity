📋 PROBLEMA ORIGINAL
═══════════════════════════════════════════════════════════════════════════════
❌ "Las alertas no se muestran cuando paso una tarjeta NFC en el dashboard"


🔍 ANÁLISIS REALIZADO
═══════════════════════════════════════════════════════════════════════════════

Se identificaron 5 causas principales:

1. ❌ AlertaController No Existe
   Faltaba: app/Http/Controllers/AlertaController.php
   Impacto: No hay endpoint para gestionar alertas

2. ❌ No Hay API de Alertas
   Faltaban: 10 rutas en routes/api.php
   Impacto: Frontend no puede obtener alertas

3. ❌ No Se Crean Alertas
   Faltaba: Lógica en CredencialController.validarNfc()
   Impacto: BD nunca se llena, no hay registros

4. ❌ No Se Cargan Alertas
   Faltaba: Función loadAlertas() en dashboard.js
   Impacto: Frontend no intenta obtener alertas

5. ❌ Panel es Estático
   Faltaba: Contenedor dinámico en index.html
   Impacto: Siempre muestra "Sistema operativo correctamente"

AlertaController::noAtendidas()

CredencialController::validarNfc()

│ POST /api/credenciales/validar-nfc                                     │
│ ├─ Validates tarjeta NFC                                              │
│ ├─ ✅ NUEVO: Si NO registrada → Alerta::create(tipo 1)               │
│ ├─ ✅ NUEVO: Si desactivada → Alerta::create(tipo 2)                │
│ ├─ ✅ NUEVO: Si expirada → Alerta::create(tipo 3)                   │
│ ├─ ✅ NUEVO: Si usuario inactivo → Alerta::create(tipo 4)           │
│ ├─ ✅ NUEVO: Si vigencia expirada → Alerta::create(tipo 5)          │
│ └─ Retorna: JSON con validación                                       │

FRONTEND (JavaScript + HTML)
───────────────────────────────────────────────────────────────────────────────

loadAlertas()
┌─────────────────────────────────────────────────────────────────────────┐
│ Ejecuta cada 5 segundos                                                │
│ ├─ GET /api/alertas/no-atendidas                                      │
│ ├─ Renderiza alertas en #alertas-container                            │
│ ├─ Cada alerta: [Tipo] [Descripción] [Hora] [Botón Atender]         │
│ └─ Actualiza UI automáticamente                                       │
└─────────────────────────────────────────────────────────────────────────┘

atenderAlerta(alertaId)
┌─────────────────────────────────────────────────────────────────────────┐
│ Ejecuta al hacer click "Atender"                                       │
│ ├─ POST /api/alertas/{id}/atender                                    │
│ ├─ Marca alerta como atendida en BD                                  │
│ ├─ Recarga lista de alertas                                          │
│ └─ Alerta desaparece del panel                                       │
└─────────────────────────────────────────────────────────────────────────┘

API ENDPOINTS (10 nuevas rutas)
───────────────────────────────────────────────────────────────────────────────

GET    /api/alertas                      → Todas las alertas
GET    /api/alertas/no-atendidas         → No atendidas (Dashboard)
GET    /api/alertas/{id}                 → Alerta específica
GET    /api/alertas/estadisticas         → Contar por tipo
POST   /api/alertas                      → Crear nueva alerta
POST   /api/alertas/{id}/atender         → Marcar como atendida
DELETE /api/alertas/{id}                 → Eliminar alerta
GET    /api/alertas/usuario/{usuarioId}  → Alertas por usuario
GET    /api/alertas/tipo/{tipoId}        → Alertas por tipo
GET    /api/alertas/fecha/{inicio}/{fin} → Alertas por rango


📊 TIPOS DE ALERTAS IMPLEMENTADOS
═══════════════════════════════════════════════════════════════════════════════

Tipo #1: Credencial no registrada
         ↪ Cuando UID NFC no existe en tabla credencial
         ↪ Acción: Usuario debe registrar la tarjeta

Tipo #2: Credencial desactivada
         ↪ Cuando la tarjeta fue desactivada manualmente
         ↪ Acción: Usuario debe reactivar tarjeta

Tipo #3: Credencial expirada
         ↪ Cuando fecha_expiracion < TODAY
         ↪ Acción: Usuario debe renovar tarjeta

Tipo #4: Usuario inactivo
         ↪ Cuando usuario.activo = 0
         ↪ Acción: Admin debe activar usuario

Tipo #5: Vigencia expirada
         ↪ Cuando usuario.vigencia_fin < TODAY
         ↪ Acción: Admin debe extender vigencia
