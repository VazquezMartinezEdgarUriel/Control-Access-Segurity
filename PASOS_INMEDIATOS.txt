## ❓ Si Algo No Funciona

### Paso 1 falló (cache:clear error)
```bash
# Verificar que PHP funciona
php --version

# Si no existe php, especificar ruta completa
c:\xampp\php\php.exe artisan cache:clear
```

### Paso 2 falló (Error creando tipos)
```bash
# Verificar tipos manualmente
mysql -u root -p AcControl -e "SELECT * FROM tipoalerta"

# Si no muestra nada, insertar manualmente en PhpMyAdmin
```

### Paso 3 falló (Test no pasa)
```bash
# Ver log de error
type storage\logs\laravel.log | tail -20

# O revisar en:
storage/logs/alertas-test-*.json
```

### Paso 4 falló (No aparece alerta en dashboard)
1. Abrir Consola del navegador (F12)
2. Tab "Network"
3. Pasar NFC
4. Buscar petición a `/api/alertas/no-atendidas`
5. Verificar HTTP status (debe ser 200)
6. Si es error, ver "Response" de la petición

## 📊 Verificaciones Rápidas

### ¿BD tiene datos?

### ¿API funciona?
```bash

```

## 📞 Soporte Rápido

| Problema | Solución |
|----------|----------|
| Cache: "Comando no encontrado" | Usar: `c:\xampp\php\php.exe artisan cache:clear` |
| MySQL: "Access denied" | Contraseña incorrecta. Intenta sin `-p` o `-p0000` |
| Test: "AlertaController no existe" | Ejecutar `php artisan cache:clear` |
| Dashboard: "No veo alertas" | Abrir F12 → Network → Pasar NFC → Buscar errores |
| Alertas: "Se crean pero no aparecen" | Actualizar página (F5) o limpiar localStorage |
